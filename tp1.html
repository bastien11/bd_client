Etude API:

Introduction:

Dans une première partie, nous allons vous présenter les demandes de services possibles pour l'API client de gitHub, nous l'utilisons sous la version 3.

Il y a beaucoup de services possible avec l'api Github tel que:

  • "Basics authentification: " qui consiste en l'apprentissage des requêtes élémentaires à l'identification et par la suite la connexion d'un utilisateurs à une API.

  • "Discovering ressource for a user: " permet à la suite de "Basics authentification" de gérer des utilisateurs possédant différents privilège au travers d'un jeton OAuth (voir https://developper.github.com/vs/guides/discovering-ressources-for-a-user/).

  • "Rendering data as graphs: "

Nous avons étudié l'API de météo sur laquelle nous avons identifer plusieurs requêtes possible:

  • On peut utiliser la méthode GET pour rechercher les prévisions météo par nom de ville sur 5 jours. La requête possède un argument 'q' qui représente le nom de la ville et le code de l'état.

  • On utilise également la methode GET pour récupérer les prévisions météo par ID de ville. On utilise ici l'agument 'id' pour l'id des villes.

Pour finir, une requête GET se caractérise par le ? a la fin de l'URL comme ceci: "api.openweathermap.org/data/2.5/forecast?" et est ensuite suivi des différents arguments qui la composant

Quelques examples de requêtes GET et POST sur l'API météo :
"GET" :

fetch("api.openweathermap.org/data/2.5/forecast?id=524901")

.then(response => response.JSON)

.then(responseJson => console.log(responseJson)

"POST" :

fetch("api.openweathermap.org/data/2.5/forecast?id=524901",
{
method: "POST"
headers: {
'Content-Type': 'application/json',
},
})
.then(response => response.JSON)
.then(responseJson => console.log(responseJson)
